Brand rules — 01

Typography

The framework uses a single typeface family throughout. Consistency and clarity over decoration.

Primary typeface

Poppins — Google Fonts — primary typeface

The quick brown fox

@import url('https://fonts.googleapis.com/css2?family=Poppins:wght@100;200;300;400;500;600;700;800;900&display=swap');

font-family: 'Poppins', Arial, Helvetica, sans-serif;

Poppins is a geometric sans-serif with a comprehensive weight range. Fallback stack: Arial, Helvetica, sans-serif. All nine weights imported.

Weight usage

300LightBody text, captions, descriptions — default reading weight
400RegularNavigation links, labels, ui text
500MediumNav links, button text, register names

Absolute rules

Rule: No serif typefaces anywhere in the framework. Poppins only. The blockquote quotation marks use a font-family override (Georgia) for the decorative glyph only — this is the sole exception and must not be extended.
Do

Poppins for all text. Sentence case for ui labels and navigation. Hierarchy through weight, scale and space only.

Don't

No serif fonts. No all-uppercase text in body copy, navigation or ui labels. No decorative or display typefaces.

Brand rules — 02

Colour palette

The palette is deliberately restrained. Two brand colours, one dark ground, one light ground. All other values are neutrals. Strict hex-only — no rgb() or rgba() in production code.

Brand colours

Saturation lock: Brand blue is fixed at #0054dc. This value is enforced with !important on the nav element to prevent inheritance corruption. Do not modify.
Do

Hex-only throughout. Use hex-8 (8-digit hex) for transparency values. Brand blue on white or stone. Brand orange for hover states, accent rules, and interactive feedback.

Don't

No rgb() or rgba() in production. No additional brand colours. Do not alter the saturation or brightness of --brand-blue.

Brand rules — 03

Voice & tone

The voice of Paul Fillingham / Thinkamigo applied consistently across all content on the framework.

What the voice is

QualityIn practice
Direct and consideredNo waffle. Say the thing. Then stop.
Warmth without sentimentalityHuman, not corporate. Engaged, not gushing.
Intellectual curiosity without jargonThe idea matters more than demonstrating you know the vocabulary.
Craft visible but worn lightlyThe work speaks. You don't need to explain it.
Serious about the work, not preciousRigour without reverence. It's allowed to be funny.
Humour earned, not performedDe Bono was right — wit is a marker of intelligence. But it must land naturally.

What to avoid

Avoid

Corporate language. Breathless enthusiasm about technology for its own sake. Over-explaining what the work is trying to do. Nostalgia as a destination (the past is load-bearing, not load-limiting). Consolation dressed as strategy.

Tech spec — 01

Architecture

The framework is built as four discrete CSS modules, each with a distinct responsibility. Hand-rolled HTML/CSS/JS — no framework dependencies.

Chassis chassis.css — v6.4

Global shell. Navigation, footer, core reset, ui components, responsive engine. Loads first on every page.

Destination engine destination.css — v4.23

Editorial content pages. The paper sheet, the text rail, typography, blockquotes, gallery grid, contact form.

Gallery engine gallery.css — v9.0

Image grid system. Lightbox, portrait grid, full-bleed breakout, resource cards, inline story images.

Theatre engine theatre.css — v5.5

Video environment. Sovereign cinema projector, 4-up grid, thumbnail system, inline video for story pages.

Load order

<!-- Always load in this order -->
<link rel="stylesheet" href="assets/css/chassis.css">
<link rel="stylesheet" href="assets/css/destination.css">  <!-- content pages -->
<link rel="stylesheet" href="assets/css/gallery.css">       <!-- if galleries present -->
<link rel="stylesheet" href="assets/css/theatre.css">       <!-- if video present -->

Page body class

<!-- Content pages require this class on body -->
<body class="content-page">
Tech spec — 02

CSS variables

All root variables are declared in chassis.css and destination.css. Destination overrides chassis where both declare the same variable — this is intentional.

Chassis :root

:root {
  --brand-blue:       #0054dc;
  --brand-blue-light: #ffffff26;  /* 15% opacity hex-8 */
  --brand-orange:     #ff8c00;
  --footer-bg:        #0a254a;
  --white:            #ffffff;
  --stone-bg:         #f5f5ef;
}

Destination :root (overrides)

:root {
  --brand-blue:   #0054dc;   /* Saturation lock — enforced */
  --brand-orange: #ff6600;   /* Destination uses #ff6600, chassis uses #ff8c00 */
  --stone-bg:     #f5f5ef;
  --charcoal:     #191919;
}
Note: Brand orange has two values across modules — #ff8c00 in chassis and #ff6600 in destination/theatre. Destination takes precedence on content pages. Resolve to a single value in next refactor.

Module — gallery engine

Gallery engine v9.0

The image presentation system. Includes lightbox, multiple grid types and the full-bleed breakout.

Lightbox

Midnight blue (#0a254a) background — synced with the theatre projector. Triple-slot filmstrip architecture (300vw track) for smooth prev/next transitions. SVG close/prev/next icons embedded as data-uris. Fixed position, z-index 9999999. Scroll locked via .no-scroll on body.

Image rules

/* All gallery images */
cursor: zoom-in;            /* Native cursor signal */
outline: 1px solid #dddddd; /* Inset border — no box model impact */
outline-offset: -1px;
transition: none;            /* No hover transitions on images */

/* Portrait grid: aspect ratio enforced */
.gallery-portrait-4 .gallery-item img {
  aspect-ratio: 2 / 3;
  object-fit: cover;
}
Module — theatre engine

Theatre engine v5.5

The sovereign cinema environment for video content. Two contexts: the full theatre grid page and inline video within story pages.

The projector

Midnight blue (#0a254a) full-screen overlay. 95vw × 16:9 stage. z-index 1000000. Synced background colour with gallery lightbox for visual consistency across media types.

Theatre grid charcoal

Charcoal (#1a1a1a) section background. 1250px container. 4-up video grid. White h1 (600 weight), 28px h2 (200 weight, #bbbbbb). Orange rule below h1.

Play icon

CSS-only triangle via border trick. White with drop-shadow keyline for visibility on light backgrounds. 0.3 base opacity, rises to 1 on hover. Scaled up for inline story context (32px vs 20px border-left width).

Usage — 02

Absolute rules

These rules are non-negotiable. They define the framework and must not be overridden.

#RuleReason
1Poppins only. No serif typefaces.Bauhaus principle — geometric purity. The framework's typographic identity.
2Sentence case throughout. No all-uppercase.Sentence case is a deliberate design decision, not a convention.
3Hex-only colour values. Hex-8 for transparency.Consistency and predictability across browsers and modules.
4--brand-blue: #0054dc is locked. Do not alter.Saturation lock enforced via !important on nav. Modification corrupts the system.
5Load chassis.css first on every page.It establishes the reset and variables that all other modules depend on.
6body.content-page required for destination engine styles to apply.Scopes destination styles to prevent bleed onto non-content pages.
7figure { margin: 0; padding: 0; } — do not override.Prevents the "stone gap" and "huge margin" browser rendering bugs.
8The text rail is 960px. Do not widen it.~96 characters per line maximum is the editorial standard.
9Gallery image transitions: none.Zoom-in cursor signals interactivity. Animation on hover is visual noise.
10Lightbox and projector share #0a254a background.Visual parity between image and video full-screen environments.
Known issue to resolve: Brand orange has two values — #ff8c00 in chassis and #ff6600 in destination and theatre. Consolidate to #ff6600 across all modules in next major version.
